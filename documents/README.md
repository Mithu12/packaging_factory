## Contributing

When adding new documentation:
1. Use clear, descriptive filenames in UPPERCASE with underscores
2. Include date and status at the top of each document
3. Keep implementation guides separate from summary documents
4. Update this README with new document categories as needed

## Maintenance

Documentation should be updated when:
- New features are implemented
- Bugs are fixed that affect documented behavior
- System architecture changes
- Integration patterns change
